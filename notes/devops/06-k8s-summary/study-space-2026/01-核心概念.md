# 01 · 核心概念

## K8s 的 10 大基础概念

### 1. 集群与节点(Cluster & Nodes)
- **集群** = **控制面(Control Plane)** + **工作节点(Worker Nodes)**。
- 控制面是"大脑":API Server、Scheduler(调度)、Controller Manager(集群级控制器)、etcd(存储)。
- 工作节点跑容器,每个节点有 kubelet + 容器运行时 + kube-proxy(负责网络转发与负载均衡)。
- 你只和控制面对话,由它决定什么跑在哪。

### 2. Pod —— 最小调度单位
- K8s 调度的最小单位是 **Pod**,不是容器。
- 一个 Pod 含 1+ 容器,**共享网络(同一 IP,用 localhost 互通)和存储卷**,但文件系统隔离。
- Pod 是**易逝的**:节点挂了或被删就没了,重建后 IP 会变。所以几乎从不直接建裸 Pod。

### 3. 声明式模型:spec vs status
| 字段 | 谁写 | 含义 |
|------|------|------|
| **spec** | 用户 | 期望状态——"我想要 3 个副本" |
| **status** | K8s | 实际状态——"现在跑着 2 个" |
- 你描述"要什么",而不是"怎么做"。`kubectl explain pods.spec` 可查任意字段文档。

### 4. 控制器与调和循环(Reconciliation Loop)★ K8s 的灵魂
- 控制器不停对比 spec 与 status,有差距就行动去消除它。
- 想要 3 个只有 2 个?→ 再起 1 个。这就是**自愈**的来源。
- GitOps(Argo CD)就是把这个循环扩展到了 Git。

### 5. 工作负载控制器
- **Deployment** 无状态应用 / **StatefulSet** 有状态 / **DaemonSet** 每节点一份 / **Job·CronJob** 一次性·定时。

### 6. Service 与网络
- **Service** 给一组飘忽的 Pod 一个稳定 IP + DNS 名,负载均衡(经 Endpoints)。
- **Ingress** 提供 L7 HTTP 路由(域名/路径)给外部流量。
- 解决的核心问题:"Pod 的 IP 一直变,怎么找到它们?"

### 7. 标签与选择器(Labels & Selectors)★ K8s 的"胶水"
- 键值标签(`app: web`、`env: prod`)+ 选择器匹配。
- Service 靠标签找 Pod,Deployment 靠标签管 Pod,调度靠标签。**松耦合靠匹配,而非硬编码引用**。
```bash
kubectl get po -l env=prod          # 有 env=prod
kubectl get po -l env               # 有 env 这个 key
kubectl get po -l '!env'            # 没有 env 标签
kubectl get po -l 'env in (prod,staging)'
kubectl delete po -l 'app=old'      # 按标签批量删
```
- **标签 vs 注解**:标签用于**筛选/选择**(短、可索引);注解(annotation)用于**存元数据**(可长、不能用于选择)。

### 8. 命名空间(Namespace)
- 把集群切成虚拟分区,用于隔离资源、配额(ResourceQuota)、限定 RBAC 范围。
- ⚠️ Namespace 只隔离**名称和资源**,**不隔离网络流量**(那是 NetworkPolicy 的事)。

### 9. ConfigMap & Secret
- 把配置从镜像解耦。ConfigMap 存普通配置,Secret 存敏感数据。
- 同一镜像挂不同配置 → 适配多环境。⚠️ Secret 默认只是 base64,**不是加密**。

### 10. 存储:Volume、PV、PVC
- 容器文件系统是易逝的。Volume 让数据跨容器重启保留。
- **PV** = 一块集群存储;**PVC** = 对存储的申领。应用只管申领,不用关心底层是 EBS/NFS/Ceph。

## 对象模型补充

### 任意对象的 4 个顶层字段
`apiVersion`、`kind`、`metadata`、`spec`(部分有 `status`)。

### 核心资源分类
| 类别 | 代表资源 |
|------|----------|
| 工作负载 Workload | Pod、Deployment、StatefulSet、DaemonSet、Job、CronJob |
| 服务发现 & 负载均衡 | Service、Ingress |
| 配置 & 存储 | ConfigMap、Secret、PV、PVC |
| 集群 Cluster | Namespace、Node、RBAC |
| 元数据 Metadata | HPA、PodDisruptionBudget、CRD |

### 自定义资源
- **CRD(Custom Resource Definition)** 让你扩展 K8s API,定义自己的资源类型;配合 Operator(控制器)实现自定义调和逻辑。
